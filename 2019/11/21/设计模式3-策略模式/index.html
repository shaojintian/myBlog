设计模式3-策略模式

发表于 2019-11-21 更新于 2019-12-12

详细代码点此链接

介绍：相对于工厂模式，不将具体的类暴露给调用者，而是在工厂中判断需要哪个类，则提供调用该类的方法给调用者。

应用场景：

优势：类的特性相同，且频繁切换类，且只需要用到单一的算法。

劣势：加入新类需要重新编译打包，无法将多个类组合使用的情况。
